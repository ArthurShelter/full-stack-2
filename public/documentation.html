myFlix

This is an API that interacts with a database that stores data about different movies.
Business Logic URL HTTP Method Request body data format Response body data format
Return a list of ALL movies to the user; /movies GET None A JSON object holding data about every movie
Return data (description, genre, director, image URL, whether it's featured or not) about a single movie by title to the user; /movies/[Title] GET None A JSON object holding data about a single movie
Return data about a genre (description) by name/title (e.g., "Thriller"); /movies/[genreName] GET None A JSON object holding data about the selected genre
Return data about a director (bio, birth year, death year) by name; /movies/[director] GET None A JSON object holding data about the selected director
Allow new users to register; /users POST A JSON object holding data about the new user to be registered A JSON object holding data about the new user that was registered, along with a confirmation message
Allow users to update their user info (username); /users/:username PUT None A confirmation message indicating the user and the data that was updated
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added - more on this later); /users/FavoriteMovies/[Title] PUT A JSON object containing the information about the movie to be added to the list of favorites. A confirmation message containing the information about the movie that was added to the favorites list
Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed - more on this later); /users/FavoriteMovies/[Title] DELETE None A confirmation message saying that the movie was removed from the list of favorites
Allow existing users to deregister (showing only a text that a user email has been removed - more on this later). /users/[ID] DELETE None A confirmation message saying that the user email was removed
